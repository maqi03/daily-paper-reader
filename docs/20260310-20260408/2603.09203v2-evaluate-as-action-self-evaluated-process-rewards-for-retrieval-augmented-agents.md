## 摘要
检索增强智能体能够查询外部证据，但其在多步推理中的可靠性仍然有限：噪声检索可能会干扰多跳问答，而仅基于结果的强化学习提供的信用分配信号过于粗糙，无法优化中间步骤。我们提出了 EvalAct (Evaluate-as-Action)，它将隐式的检索质量评估转化为显式动作，并强制执行一种耦合的“搜索-评估”协议，使得每次检索后立即跟随一个结构化的评估分数，从而产生与交互轨迹对齐的过程信号。为了利用这些信号，我们引入了过程校准优势重缩放 (PCAR)，这是一种基于 GRPO 的优化方法，它根据评估分数在片段级别对优势值进行重缩放，在强调可靠片段的同时保守地更新不确定片段。在七个开放域问答基准测试上的实验表明，EvalAct 实现了最佳的平均准确率，在多跳任务上的提升最为显著；消融实验验证了显式评估循环是主要的改进驱动力，而 PCAR 则提供了持续的额外收益。

## Abstract
Retrieval-augmented agents can query external evidence, yet their reliability in multi-step reasoning remains limited: noisy retrieval may derail multi-hop question answering, while outcome-only reinforcement learning provides credit signals that are too coarse to optimize intermediate steps. We propose \textsc{EvalAct} (Evaluate-as-Action), which converts implicit retrieval quality assessment into an explicit action and enforces a coupled Search-to-Evaluate protocol so that each retrieval is immediately followed by a structured evaluation score, yielding process signals aligned with the interaction trajectory. To leverage these signals, we introduce Process-Calibrated Advantage Rescaling (PCAR), a GRPO-based optimization method that rescales advantages at the segment level according to evaluation scores, emphasizing reliable segments while updating uncertain ones conservatively. Experiments on seven open-domain QA benchmarks show that \textsc{EvalAct} achieves the best average accuracy, with the largest gains on multi-hop tasks, and ablations verify that the explicit evaluation loop drives the primary improvements while PCAR provides consistent additional benefits.

### 1. 核心问题与整体含义
论文针对**检索增强智能体（RAG Agents）**在处理复杂多步推理（如多跳问答）时的两大瓶颈：
*   **错误传播（Error Propagation）**：在多步交互中，一次无关或含噪声的检索会误导后续推理，导致轨迹偏离，且模型缺乏显式的自我纠正机制。
*   **信用分配粗糙（Coarse Credit Assignment）**：现有的强化学习（RL）方法（如 GRPO）通常只依赖最终答案的正确性（Outcome Reward）。这种稀疏信号无法区分长轨迹中哪些检索步骤是有益的，哪些是冗余或有害的。

**核心含义**：论文提出将“评估检索质量”这一隐式行为转化为显式的**策略动作**，通过构建“搜索-评估”闭环，为 RL 提供细粒度的过程信号。

### 2. 方法论：EvalAct 与 PCAR
论文提出了两个核心组件：

*   **EvalAct (Evaluate-as-Action) 框架**：
    *   **显式动作化**：将评估定义为一种动作 `Evaluate(c, z)`，其中 `c` 是文本分析，`z` 是 0-10 的置信度评分。
    *   **耦合协议**：强制执行 `Search → Evaluate` 协议。每次执行搜索后，智能体必须立即对检索到的证据进行自我评估。
    *   **环境反馈**：环境根据评分 `z` 返回离散的控制指令（如“质量高，请继续”或“质量低，请重新搜索”），在推理阶段实现早期剪枝。

*   **PCAR (Process-Calibrated Advantage Rescaling) 优化算法**：
    *   **基于 GRPO 的改进**：在标准的组相对策略优化（GRPO）基础上，利用智能体自产的评分 `z` 对优势值（Advantage）进行重缩放。
    *   **片段级缩放**：计算轨迹内评分的标准化可靠性信号，对高分（可靠）的检索片段放大梯度更新，对低分（不确定）的片段进行保守更新。
    *   **公式逻辑**：通过一个缩放因子 $\lambda$ 调整 token 级别的优势值 $\hat{A}_{i,t}$，从而在不需要昂贵的人工标注过程奖励模型（PRM）的情况下，实现过程级指导。

*   **性能领先**：EvalAct 在所有基准测试中取得了最佳平均准确率（3B 模型 44.0%，7B 模型 47.1%）。
*   **多跳优势显著**：在复杂的多跳任务上提升最明显（如在 2WikiMultihopQA 上比最强基线高出约 10 个百分点），证明了显式评估对长程推理的价值。
*   **结构胜过算法**：消融实验显示，**显式评估循环（Eval Loop）**是性能提升的核心驱动力（贡献了约 7.5% 的增益），而 PCAR 算法提供了额外的 1.2% 增益。
*   **SFT 的必要性**：直接进行 EvalAct RL 效果较差，必须先通过少量 SFT 让模型学会“搜索-评估”的格式规范。

### 7. 优点
*   **无需人工 PRM**：通过将评估转化为动作，巧妙地利用模型自身的置信度作为过程奖励，降低了训练成本。
*   **推理可控性**：在推理时，评估分数可以直接转化为控制指令，增强了智能体在面对噪声时的鲁棒性。
*   **即插即用**：PCAR 算法可以作为现有 GRPO 框架的插件，具有较好的通用性。

### 8. 不足与局限
*   **自评估偏差风险**：方法依赖于模型的自我评估能力。如果模型存在严重的过度自信或校准不良，PCAR 可能会强化错误的路径（尽管文中提到 PCAR 机制能一定程度抑制奖励作弊）。
*   **推理开销增加**：由于每个搜索步骤后都强制增加了一个评估动作，推理时的 Token 消耗和延迟会比普通 RAG 更高。
*   **任务局限**：目前主要在问答任务上验证，对于更复杂的开放式工具调用场景（如操作数据库或 API 序列）的泛化性有待进一步研究。
