## 摘要
计算机使用智能体（CUAs）利用大语言模型在桌面环境中执行图形用户界面（GUI）操作，但它们在生成动作时往往不评估动作质量，导致不可逆的错误在后续步骤中级联。我们提出了 IntentScore，这是一种具备规划感知能力的奖励模型，它通过学习跨越三个操作系统的 398K 条离线 GUI 交互步骤来对候选动作进行评分。IntentScore 采用两个互补的目标进行训练：用于状态-动作相关性的对比对齐和用于动作正确性的边际排序。在架构上，它将每个候选动作的规划意图嵌入到动作编码器中，从而能够区分动作相似但基本原理不同的候选动作。IntentScore 在留出法评估中实现了 97.5% 的成对判别准确率。在训练过程中完全未见的 OSWorld 环境中，IntentScore 作为 Agent S3 的重排序器部署，将任务成功率提升了 6.9 个百分点，证明了从异构离线轨迹中学习到的奖励估计可以泛化到未见的智能体和任务分布。

## Abstract
Computer-Use Agents (CUAs) leverage large language models to execute GUI operations on desktop environments, yet they generate actions without evaluating action quality, leading to irreversible errors that cascade through subsequent steps. We propose IntentScore, a plan-aware reward model that learns to score candidate actions from 398K offline GUI interaction steps spanning three operating systems. IntentScore trains with two complementary objectives: contrastive alignment for state-action relevance and margin ranking for action correctness. Architecturally, it embeds each candidate's planning intent in the action encoder, enabling discrimination between candidates with similar actions but different rationales. IntentScore achieves 97.5% pairwise discrimination accuracy on held-out evaluation. Deployed as a re-ranker for Agent S3 on OSWorld, an environment entirely unseen during training, IntentScore improves task success rate by 6.9 points, demonstrating that reward estimation learned from heterogeneous offline trajectories generalizes to unseen agents and task distributions.

### 1. 核心问题与研究背景
*   **核心问题**：计算机使用智能体（CUA）在执行复杂的图形用户界面（GUI）任务时，通常采用“单次生成”模式，缺乏对动作质量的评估。一旦产生错误动作，由于 GUI 环境状态往往不可逆，错误会迅速累积导致任务失败。
*   **研究背景**：现有的缓解方案（如 LLM 自评估或环境回滚）存在推理成本高、自偏见（Self-preference bias）或在现实延迟约束下不可行等问题。因此，需要一种轻量级、独立且具备泛化能力的奖励模型来实时筛选最优动作。

### 2. 方法论：IntentScore 架构与训练
*   **核心思想**：利用大规模离线 GUI 交互轨迹，训练一个双编码器（Dual-encoder）奖励模型。该模型不仅考虑当前状态和动作，还特别引入了**规划意图（Planning Intent）**作为条件，以区分表面相似但动机不同的动作。
*   **关键技术细节**：
    *   **双编码器架构**：
        *   **状态编码器（State Encoder）**：融合当前截图（SigLIP 编码）、文本上下文（任务指令、观察、反思）以及通过 GRU 总结的最近 3 步历史信息。
        *   **意图感知动作编码器（Action Encoder）**：将候选动作的描述、执行代码、坐标与**智能体的思考过程（Thought/Intent）**共同编码。
    *   **双目标训练函数**：
        1.  **对比对齐（InfoNCE）**：学习状态与动作的相关性，即“在当前界面下，这个动作是否合理”。
        2.  **边际排名（Margin Ranking）**：通过硬负采样（如同一任务中相邻步骤的动作或标注为错误的动作），强制拉开正确动作与错误动作的得分差距。
    *   **两阶段流水线**：先在跨系统（Windows, Mac, Ubuntu）的 398K 步数据上进行预训练，再在目标域（Ubuntu）上进行微调。

### 6. 主要结论与发现
*   **性能提升**：在 OSWorld 基准测试中，IntentScore 将 Agent S3 的任务成功率提升了 **6.9 个百分点**（从 45.2% 升至 52.1%）。
*   **意图的关键作用**：将规划意图引入动作编码器使判别准确率提升了 4.7%，证明了“知道为什么要这么做”对评估动作质量至关重要。
*   **训练目标的互补性**：单纯的对比对齐（Alignment）无法区分正确与错误动作（得分差距仅 0.002），必须结合边际排名损失（得分差距扩大至 0.046）。
*   **跨系统泛化**：尽管 UI 视觉差异巨大，但从 Windows 和 Mac 数据中学习到的抽象 GUI 模式能显著增强在 Ubuntu 上的表现。

### 7. 优点
*   **高效性**：模型极小且推理极快，适合集成到实时交互系统中。
*   **意图条件化**：巧妙地解决了 GUI 动作中“语义重叠”的问题（即同一个点击动作在不同意图下可能是对或错）。
*   **鲁棒性**：通过文本增强技术，模型对不同 LLM 生成的动作描述风格具有较强的鲁棒性。

### 8. 不足与局限
*   **分布偏移（Distribution Shift）**：离线训练数据（人工标注或特定模型生成）与在线部署时智能体生成的动作在格式和风格上存在差异，导致离线高准确率（97.5%）到在线成功率提升（6.9%）之间存在折损。
*   **视觉编码器限制**：使用冻结的视觉编码器可能无法捕捉极细微的 UI 变化，这在某些需要高精度定位的任务（如 Impress 绘图）中表现为性能瓶颈。
*   **单步评估**：目前模型仅针对单步动作进行评分，尚未实现多步规划的长期价值估计。
