## 摘要
组相对策略优化 (GRPO) 广泛应用于具有可验证奖励的强化学习，但它经常面临优势坍缩（advantage collapse）的问题：当一个组内的所有采样序列（rollouts）获得相同的奖励时，该组产生的相对优势为零，从而无法提供学习信号。例如，如果一个问题对推理器而言过于困难，所有采样的序列可能都是错误的并获得零奖励。最近的研究通过为此类难题添加提示或辅助支架来解决这一问题，使推理器能够产生混合的结果并恢复非零的更新信号。然而，现有的提示通常是固定的，而非自适应于当前的推理器；此外，在有提示输入下产生学习信号的提示并不一定能改进测试时使用的无提示策略。为此，我们提出了强化学习中的提示学习（HiLL），这是一个在强化学习过程中联合训练提示器策略和推理器策略的框架。对于每个难题，提示器会根据当前推理器的错误采样序列在线生成提示，从而使提示生成能够适应推理器不断演变的错误。我们进一步引入了提示依赖度（hint reliance），用于衡量正确的提示轨迹对提示的依赖程度。我们推导出了一个可迁移性结果，表明较低的提示依赖度意味着从有提示成功到无提示成功的迁移能力更强，并利用该结果定义了用于训练提示器的迁移加权奖励。因此，HiLL 倾向于选择那些不仅能恢复信息丰富的 GRPO 组，还能产生更有可能改进原始无提示策略信号的提示。在多个基准测试上的实验表明，HiLL 一致优于 GRPO 和以往基于提示的基准方法，证明了自适应且具备迁移感知的提示学习在强化学习中的价值。代码见 https://github.com/Andree-9/HiLL。

## Abstract
Group Relative Policy Optimization (GRPO) is widely used for reinforcement learning with verifiable rewards, but it often suffers from advantage collapse: when all rollouts in a group receive the same reward, the group yields zero relative advantage and thus no learning signal. For example, if a question is too hard for the reasoner, all sampled rollouts can be incorrect and receive zero reward. Recent work addresses this issue by adding hints or auxiliary scaffolds to such hard questions so that the reasoner produces mixed outcomes and recovers a non-zero update. However, existing hints are usually fixed rather than adapted to the current reasoner, and a hint that creates learning signal under the hinted input does not necessarily improve the no-hint policy used at test time. To this end, we propose Hint Learning for Reinforcement Learning (HiLL), a framework that jointly trains a hinter policy and a reasoner policy during RL. For each hard question, the hinter generates hints online conditioned on the current reasoner's incorrect rollout, allowing hint generation to adapt to the reasoner's evolving errors. We further introduce hint reliance, which measures how strongly correct hinted trajectories depend on the hint. We derive a transferability result showing that lower hint reliance implies stronger transfer from hinted success to no-hint success, and we use this result to define a transfer-weighted reward for training the hinter. Therefore, HiLL favors hints that not only recover informative GRPO groups, but also produce signals that are more likely to improve the original no-hint policy. Experiments across multiple benchmarks show that HiLL consistently outperforms GRPO and prior hint-based baselines, demonstrating the value of adaptive and transfer-aware hint learning for RL. The code is available at https://github.com/Andree-9/HiLL.

### 1. 核心问题与研究动机
*   **核心问题**：在具有可验证奖励的强化学习（RLVR）中，广泛使用的 **GRPO（组相对策略优化）** 算法存在“优势坍缩”（Advantage Collapse）现象。当模型面对难题时，一组内的所有采样序列（rollouts）可能全部错误，导致相对优势为零，模型无法获得任何学习信号。
*   **研究背景**：现有方法尝试通过添加固定提示（Hints）或辅助支架来诱导正确答案，但存在两个痛点：
    1.  **非自适应性**：提示通常是预设的，无法根据推理器当前的错误模式动态调整。
    2.  **迁移鸿沟**：在有提示下学到的成功经验，如果不加约束，往往会变成对提示的“依赖”（如提示直接给出了关键步骤），导致模型在测试（无提示）时依然无法独立解题。

### 2. 方法论：HiLL 框架
HiLL（Hint Learning for RL）的核心思想是**联合训练一个提示器（Hinter）和一个推理器（Reasoner）**。
*   **失败驱动的在线提示生成**：
    *   当推理器在某题上全组失败时，提示器介入。
    *   提示器的输入包括：原题、推理器刚才失败的推理路径、参考答案。这使得提示器能针对推理器的具体错误点进行“因材施教”。
*   **提示依赖度（Hint Reliance）**：
    *   定义公式：$\rho(\tau; q, h) = \log \pi_\theta(\tau | q+h) - \log \pi_\theta(\tau | q)$。
    *   该指标衡量一条正确的推理路径 $\tau$ 在有提示和无提示情况下的对数概率差。差值越大，说明模型越依赖提示。
*   **转移加权奖励（Transfer-Weighted Reward）**：
    *   提示器的奖励由两部分组成：**信号创建**（使推理器产生对错混合的样本组）和**信号迁移**（基于提示依赖度的指数衰减项）。
    *   公式：$R = \text{Signal Creation} \times \exp(-\max(\hat{\rho}_c, 0) / T)$。
    *   该奖励机制迫使提示器生成“启发式”而非“灌输式”的提示。
*   **联合优化流程**：
    1.  推理器采样，识别全错组。
    2.  提示器生成多个候选提示。
    3.  推理器在提示下重新采样，计算奖励。
    4.  利用 GRPO 同时更新推理器（学习解题）和提示器（学习如何更好地辅助）。

### 6. 主要结论与发现
*   **性能提升**：HiLL 在所有数学和通用推理基准上均显著优于标准 GRPO 和其他提示基线。
*   **迁移性验证**：引入“提示依赖度”约束后，模型生成的提示更短、更具概念性（Conceptual），而非直接给出计算步骤。
*   **解决坍缩**：HiLL 有效降低了训练过程中的“全错组”比例，将原本无用的难题转化为有效的训练信号。
*   **理论支撑**：论文证明了较低的提示依赖度能为无提示下的成功概率提供更高的下界。

### 7. 优点
*   **理论与实践结合**：提出了提示依赖度的数学定义，并推导了迁移边界，为提示学习提供了理论依据。
*   **动态演进**：提示器随推理器的进步而进化，能够始终针对推理器的“能力边界”提供帮助。
*   **高质量信号**：通过奖励函数设计，避免了提示变成“作弊工具”，确保了学习到的知识能迁移到测试场景。

### 8. 不足与局限
*   **训练开销**：由于需要额外的提示生成和二次采样步骤，训练时间显著增加（2.6x - 3.8x），算力成本较高。

*   **任务局限**：目前主要集中在具有明确验证逻辑（Verifiable Rewards）的任务（如数学、代码），对于主观性较强的文本生成任务适用性尚待验证。
