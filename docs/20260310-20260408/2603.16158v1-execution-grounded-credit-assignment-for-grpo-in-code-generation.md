## 摘要
无评论员（Critic-free）的带可验证奖励的强化学习（RLVR）通过优化单元测试通过率来改进代码生成，但 GRPO 风格的更新面临粗粒度的信用分配问题：即使失败源于局部的语义错误，单一的结果信号也会均匀地分布在长程序中。我们提出了基于执行的信用分配（EGCA），它利用执行轨迹使 GRPO 更新局部化。对于满足算法约束但未通过测试的程序，EGCA 在相同的插桩环境下执行候选方案和标准参考解（离线整理一次；用于分析而非监督），识别最早的语义分歧点，并仅对相应的 token 跨度分配优势值，同时屏蔽下游 token。EGCA 是一种即插即用的修改，不需要评论员、辅助损失或学习得到的验证器，在 HumanEval 上实现了 82.1% 的 pass@1（比 GRPO 提高 3.1%），在 MBPP 上实现了 68.9%（提高 1.5%），且仅有 18% 的实际运行时间开销。

## Abstract
Critic-free reinforcement learning with verifiable rewards (RLVR) improves code generation by optimizing unit-test pass rates, but GRPO-style updates suffer from coarse credit assignment: a single outcome signal is spread uniformly across long programs even when failure stems from a localized semantic error. We propose Execution-Grounded Credit Assignment (EGCA), which localizes GRPO updates using execution traces. For programs that satisfy algorithmic constraints but fail tests, EGCA executes the candidate and a canonical reference solution (curated once offline; used for analysis, not supervision) under identical instrumentation, identifies the earliest semantic divergence, and assigns advantage only to the corresponding token span while masking downstream tokens. EGCA is a drop-in modification requiring no critic, auxiliary loss, or learned verifier, yielding 82.1% pass@1 on HumanEval (+3.1 over GRPO) and 68.9% on MBPP (+1.5) with 18% wall-clock overhead.

### 1. 核心问题与研究背景
*   **研究动机**：在代码生成任务中，基于可验证奖励的强化学习（RLVR，如 GRPO）通常将单元测试的二元结果（通过/失败）均匀地分配给整个代码序列。
*   **核心痛点**：这种“信用抹杀”（Credit Smear）现象导致即使代码中只有一个微小的逻辑错误（如符号写反），整个正确的代码框架也会受到惩罚。随着模型能力的提升，代码大多能通过编译且结构合理，失败往往源于局部的语义偏差，因此精准的**信用分配（Credit Assignment）**成为进一步提升性能的关键瓶颈。

### 2. 方法论：EGCA (Execution-Grounded Credit Assignment)
*   **核心思想**：利用代码的可执行性，通过对比候选代码与参考代码在执行过程中的状态变化，定位“最早的语义分歧点”，并将强化学习的梯度更新集中在导致错误的特定 Token 跨度上。
*   **关键流程**：
    1.  **约束引导采样**：从参考解中提取算法约束（如时间复杂度、数据结构），引导模型生成结构相似的代码。
    2.  **故障模式分类**：将生成的代码分为四类：语法错误（SYNTAX）、违反约束（CONSTRAINT）、逻辑错误（LOGIC）和完全正确（CORRECT）。
    3.  **执行追踪对比**：对于 LOGIC 类代码，在相同输入下运行候选代码和参考代码，记录每一步的状态轨迹（Trace）。
    4.  **定位分歧点**：识别两者状态首次不一致的执行边界 $k^*$。
    5.  **局部化优势分配**：在 GRPO 更新中，仅对分歧点对应的 Token 跨度分配优势值（Advantage），并屏蔽（Mask）该点之后的所有 Token，避免对后续无关代码的错误惩罚。

### 3. 实验设计
*   **数据集**：训练使用 **APPS+**（包含自然语言描述、单元测试和参考解）。
*   **基准测试（Benchmark）**：在 **HumanEval** 和 **MBPP** 上评估 pass@1 准确率。
*   **对比方法**：
    *   基础模型：DeepSeek-Coder-Instruct-6.7B。
    *   强化学习基线：Vanilla PPO, GRPO, RLTF（多粒度反馈）, StepCoder（屏蔽未执行代码）, CodeRL+（辅助执行对齐损失）。

### 6. 主要结论与发现
*   **性能提升**：EGCA 在 HumanEval 上达到 82.1%（比 GRPO 提升 3.1%），在 MBPP 上达到 68.9%（提升 1.5%）。
*   **信用分配是关键**：实验证明，在模型已经能生成基本正确代码的阶段，信用分配比解决奖励稀疏问题更重要。
*   **低开销**：相比于传统的 GRPO，EGCA 仅增加了 18% 的运行时间开销，且不需要额外的 Critic 网络或复杂的辅助损失函数。

### 7. 优点与亮点
*   **无需 Critic**：保持了 GRPO 架构的简洁性，不需要训练价值网络，降低了显存占用。
*   **确定性引导**：利用编译器和执行追踪提供确定性的反馈，比学习得到的奖励模型（PRM）更可靠。
*   **即插即用**：作为一种 Token 级别的掩码策略，可以轻松集成到现有的 RLVR 框架中。

### 8. 不足与局限
*   **依赖参考解**：该方法需要离线提供标准参考代码，这限制了其在完全开放式、无标准答案的生成任务中的应用。
*   **阶段依赖性**：EGCA 在模型已经具备一定基础能力（“近乎正确”阶段）时效果最好。如果模型生成的代码连编译都无法通过，该方法的优势将无法体现。
*   **调试器能力限制**：虽然实验证明弱调试器也能提供帮助，但如果调试器无法正确解析执行轨迹或代码结构，定位信号的质量会下降。
