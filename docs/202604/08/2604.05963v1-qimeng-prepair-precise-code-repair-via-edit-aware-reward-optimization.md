## 摘要
大语言模型（LLMs）在程序修复方面表现出色，但往往受到过度编辑（over-editing）的影响，即过度的修改会覆盖正确的代码并阻碍缺陷定位。我们系统地量化了其影响，并引入了精确修复任务，该任务在仅修复有缺陷部分的同时，最大限度地重用正确代码。基于这一见解，我们提出了 PRepair 框架，旨在减轻过度编辑并提高修复准确性。PRepair 包含两个组件：Self-Breaking，通过受控的缺陷注入和极大极小采样生成多样化的缺陷程序；以及 Self-Repairing，利用编辑感知奖励，通过编辑感知群体相对策略优化（EA-GRPO）训练模型，以鼓励最小化且正确的编辑。实验表明，在综合考虑修复正确性和程度的指标 fix_1@1 下，PRepair 将修复精度提高了高达 31.4%，并且在结合投机编辑（speculative editing）时显著提升了解码吞吐量，证明了其在精确且实用的代码修复方面的潜力。

## Abstract
Large Language Models (LLMs) achieve strong program repair performance but often suffer from over-editing, where excessive modifications overwrite correct code and hinder bug localization. We systematically quantify its impact and introduce precise repair task, which maximizes reuse of correct code while fixing only buggy parts. Building on this insight, we propose PRepair, a framework that mitigates over-editing and improves repair accuracy. PRepair has two components: Self-Breaking, which generates diverse buggy programs via controlled bug injection and min-max sampling, and Self-Repairing, which trains models with Edit-Aware Group Relative Policy Optimization (EA-GRPO) using an edit-aware reward to encourage minimal yet correct edits. Experiments show that PRepair improves repair precision by up to 31.4% under $\mathrm{fix}_1@1$, a metric that jointly considers repair correctness and extent, and significantly increases decoding throughput when combined with speculative editing, demonstrating its potential for precise and practical code repair.

*   **背景**：现有模型在修复代码时，往往倾向于重写大段代码，即使只需修改一行。这种行为不仅掩盖了真实的 Bug 位置，还破坏了原始代码的逻辑结构，增加了开发者的代码评审（Review）负担。
*   **核心问题**：如何在保证修复正确性的前提下，实现“精确修复”——即最大限度重用原有的正确逻辑，仅对缺陷部分进行最小化修改。

### 2. 方法论：PRepair 框架
PRepair 框架由两个核心阶段组成，旨在解决精确修复中的数据稀缺和正确代码保留问题：

*   **Self-Breaking（自我破坏/缺陷注入）**：
    *   **核心思想**：利用 LLM 在高质量的正确代码（Golden Code）中主动注入 Bug，构建大规模的“缺陷-修复”对。
    *   **关键技术**：采用**极大极小采样（Min-Max Sampling）**策略。通过最小化样本间的成对相似度（基于编辑距离），确保生成的缺陷程序在空间分布上尽可能多样，避免模型过拟合于单一的 Bug 模式。
*   **Self-Repairing（自我修复/强化学习）**：
    *   **核心算法**：提出**编辑感知组相对策略优化（EA-GRPO）**。
    *   **奖励机制**：引入**动态编辑感知奖励（Edit-Aware Reward）**。该奖励并非简单惩罚修改，而是设置了一个“组准确率阈值 $\alpha$”。只有当模型生成的候选解在组内达到足够的正确率时，才会触发编辑惩罚。
    *   **公式逻辑**：奖励函数结合了功能正确性（是否通过单元测试）和编辑成本（Levenshtein 距离）。通过 Sigmoid 函数对编辑惩罚进行归一化，鼓励模型在保持正确的同时减少冗余修改。
