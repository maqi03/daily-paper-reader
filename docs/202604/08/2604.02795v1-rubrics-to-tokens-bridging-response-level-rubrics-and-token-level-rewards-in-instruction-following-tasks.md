## 摘要
基于评分细则（Rubric）的强化学习（RL）已成为将大语言模型（LLM）与复杂的开放域指令遵循任务对齐的一种极具前景的方法。然而，现有方法主要依赖于回复级奖励，这引入了严重的奖励稀疏性和奖励歧义问题。为了解决这些问题，我们提出了 Rubrics to Tokens (RTT)，这是一种新型的基于评分细则的强化学习框架，它桥接了粗粒度的回复级评分和细粒度的 Token 级信用分配。RTT 引入了一个 Token 级相关性判别器，用于预测回复中的哪些 Token 负责特定的约束，并通过 RTT-GRPO 优化策略模型，该算法在一个统一的框架内集成了回复级和 Token 级的优势（Advantage）。此外，在 Token 级基于评分细则的强化学习中，当从一维结果级奖励过渡到三维奖励空间时，我们提出了一种名为“样本内 Token 组归一化”（Intra-sample Token Group Normalization）的新型组归一化方法，以适应这种转变。大量的实验和基准测试表明，RTT 在不同模型中的指令级和评分细则级准确度上均持续优于其他基准方法。

## Abstract
Rubric-based Reinforcement Learning (RL) has emerged as a promising approach for aligning Large Language Models (LLMs) with complex, open-domain instruction following tasks. However, existing methods predominantly rely on response-level rewards, introducing severe reward sparsity and reward ambiguity problems. To address these issues, we propose Rubrics to Tokens (RTT), a novel rubric-based RL framework that bridges coarse response-level scores and fine-grained token-level credit assignment. RTT introduces a Token-Level Relevance Discriminator to predict which tokens in the response are responsible for a specific constraint, and optimizes the policy model via RTT-GRPO, which integrates response-level and token-level advantages within a unified framework. Furthermore, when transitioning from one-dimensional, outcome-level reward to three-dimensional reward space in the token-level rubric-based RL, we propose a novel group normalization method, called Intra-sample Token Group Normalization, to accommodate this shift. Extensive experiments and benchmarks demonstrate that RTT consistently outperforms other baselines in both instruction- and rubric-level accuracy across different models.

### 论文总结：Rubrics to Tokens (RTT) —— 桥接回复级评分与 Token 级奖励

#### 1. 核心问题与整体含义（研究动机和背景）
在大型语言模型（LLM）的对齐过程中，**指令遵循（Instruction Following）**任务通常涉及多个复杂的约束（如格式、风格、内容禁忌等）。现有的基于评分细则（Rubric-based）的强化学习方法主要依赖**回复级（Response-level）奖励**，即对整个回复给出一个综合得分。
这种做法存在两个核心痛点：
*   **奖励稀疏性（Reward Sparsity）：** 如果采用“全对才给分”的严格标准（AON），模型在训练初期很难通过随机探索获得正向信号。
*   **奖励歧义性（Reward Ambiguity）：** 如果采用“按满足比例给分”的标准（CSR），两个完全不同的错误回复可能得到相同的分数，导致模型无法精准识别哪些 Token 导致了失败，即**信用分配（Credit Assignment）**失效。

**RTT 的核心目标**是打破这种粗粒度的限制，将回复级的评分细则精准地映射到具体的 Token 上，实现细粒度的策略优化。

#### 2. 方法论：核心思想与关键技术
RTT 框架由三个关键组件构成：

*   **Token 级相关性判别器（Token-Level Relevance Discriminator）：**
    *   **核心思想：** 训练一个轻量级判别器 $D_\phi$，输入（指令、约束、回复），预测回复中每个 Token 与该约束的相关性概率 $p_t$。
    *   **数据构建：** 利用强模型（如 GPT-5.2、DeepSeek-V3）对训练数据进行标注，识别出哪些文本片段直接负责满足或违反了特定约束。
*   **RTT-GRPO 算法：**
    *   在 DeepSeek 提出的 GRPO 算法基础上进行扩展。其优势函数（Advantage）由两部分加权组成：
        $$\hat{A}_{sum} = \alpha \hat{A}_{res} + \beta \hat{A}_{tok}$$
    *   $\hat{A}_{res}$ 是传统的回复级优势，用于在不同采样样本间做对比。
    *   $\hat{A}_{tok}$ 是 Token 级优势，利用判别器的相关性得分，将约束的满足/违反信号分配到具体的 Token 上。
*   **样本内 Token 组归一化（Intra-sample Token Group Normalization）：**
    *   **解决组划分问题（Group Partitioning Problem）：** 在三维奖励空间（样本、Token、约束）中，传统的跨样本归一化会导致“长度偏差”（长回复主导均值和方差）。
    *   **技术细节：** RTT 在单个回复内部对 Token 奖励进行归一化，从而解耦了 Token 级信号与回复级信号，确保了训练的稳定性。
