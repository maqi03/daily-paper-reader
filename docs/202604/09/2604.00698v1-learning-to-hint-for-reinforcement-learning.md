## 摘要
组相对策略优化（GRPO）广泛应用于具有可验证奖励的强化学习，但它经常面临优势坍缩（advantage collapse）的问题：当一个组内的所有采样序列（rollouts）获得相同的奖励时，该组产生的相对优势为零，从而无法提供学习信号。例如，如果一个问题对于推理器而言太难，所有采样的序列可能都是错误的并获得零奖励。最近的研究通过为这类难题添加提示或辅助支架（auxiliary scaffolds）来解决这一问题，使推理器产生混合的结果并恢复非零更新。然而，现有的提示通常是固定的，而非适配当前的推理器；此外，在有提示输入下产生学习信号的提示，并不一定能改进测试时使用的无提示策略。为此，我们提出了强化学习提示学习（HiLL），这是一个在强化学习过程中联合训练提示器策略和推理器策略的框架。对于每个难题，提示器会根据当前推理器的错误采样序列在线生成提示，使提示生成能够适配推理器不断演变的错误。我们进一步引入了提示依赖度（hint reliance），用于衡量正确的带提示轨迹对提示的依赖程度。我们推导出了一个可迁移性结果，表明较低的提示依赖度意味着从有提示成功到无提示成功的迁移能力更强，并利用该结果定义了用于训练提示器的迁移加权奖励。因此，HiLL 倾向于产生不仅能恢复具有信息量的 GRPO 组，还能产生更有可能改进原始无提示策略信号的提示。在多个基准测试上的实验表明，HiLL 一贯优于 GRPO 和之前的基于提示的基准方法，证明了适配性和迁移感知提示学习在强化学习中的价值。

## Abstract
Group Relative Policy Optimization (GRPO) is widely used for reinforcement learning with verifiable rewards, but it often suffers from advantage collapse: when all rollouts in a group receive the same reward, the group yields zero relative advantage and thus no learning signal. For example, if a question is too hard for the reasoner, all sampled rollouts can be incorrect and receive zero reward. Recent work addresses this issue by adding hints or auxiliary scaffolds to such hard questions so that the reasoner produces mixed outcomes and recovers a non-zero update. However, existing hints are usually fixed rather than adapted to the current reasoner, and a hint that creates learning signal under the hinted input does not necessarily improve the no-hint policy used at test time. To this end, we propose Hint Learning for Reinforcement Learning (HiLL), a framework that jointly trains a hinter policy and a reasoner policy during RL. For each hard question, the hinter generates hints online conditioned on the current reasoner's incorrect rollout, allowing hint generation to adapt to the reasoner's evolving errors. We further introduce hint reliance, which measures how strongly correct hinted trajectories depend on the hint. We derive a transferability result showing that lower hint reliance implies stronger transfer from hinted success to no-hint success, and we use this result to define a transfer-weighted reward for training the hinter. Therefore, HiLL favors hints that not only recover informative GRPO groups, but also produce signals that are more likely to improve the original no-hint policy. Experiments across multiple benchmarks show that HiLL consistently outperforms GRPO and prior hint-based baselines, demonstrating the value of adaptive and transfer-aware hint learning for RL. The code is available at https://github.com/Andree-9/HiLL.

### 1. 核心问题与整体含义（研究动机和背景）
论文聚焦于大语言模型在强化学习（RL）中的**优势坍缩（Advantage Collapse）**问题。在使用组相对策略优化（GRPO）进行可验证奖励的 RL 训练时，如果一个问题太难，模型生成的所有采样（rollouts）可能全部错误，导致组内相对优势为零，模型无法获得学习信号。

虽然现有研究尝试通过添加“提示（Hints）”或“支架（Scaffolds）”来引导模型产生正确答案以恢复信号，但存在两个痛点：
*   **缺乏自适应性**：提示通常是静态的或离线生成的，无法随推理器能力的提升而动态调整。
*   **缺乏迁移性**：如果提示过于直接（如直接给出关键公式），模型可能学会了“走捷径”而非真正的推理，导致在测试阶段（无提示）性能提升有限。

### 2. 方法论：HiLL 框架
论文提出了 **HiLL (Hint Learning for Reinforcement Learning)**，其核心思想是**联合训练一个提示器（Hinter）和一个推理器（Reasoner）**。

*   **失败驱动的在线提示生成**：当推理器在某个问题上全错时，提示器会根据“原题 + 推理器的错误尝试 + 参考答案”在线生成多个候选提示。这使得提示器能针对推理器的具体错误模式进行“因材施教”。
*   **提示依赖度（Hint Reliance）**：为了衡量提示的质量，论文定义了提示依赖度 $\rho$。其公式为：
    $$\rho(\tau; q, h) = \log \pi_\theta(\tau | q+h) - \log \pi_\theta(\tau | q)$$
    该指标衡量了正确轨迹 $\tau$ 在有提示 ($q+h$) 和无提示 ($q$) 条件下的对数概率差。
*   **迁移加权奖励（Transfer-Weighted Reward）**：提示器的奖励函数由两部分组成：
    1.  **信号创建（Signal Creation）**：奖励那些能让推理器产生“有对有错”混合结果的提示。
    2.  **信号迁移（Signal Transfer）**：通过 $e^{-\rho}$ 对奖励进行加权，惩罚那些让模型产生高度依赖（即难以迁移到无提示场景）的提示。
*   **联合优化流程**：推理器通过 GRPO 更新以提升推理能力；提示器通过 GRPO 更新以生成更具启发性、更易迁移的提示。

### 6. 主要结论与发现
*   **性能领先**：HiLL 在所有数学和通用推理基准上均优于标准 GRPO 和现有的提示基准方法。
*   **迁移的重要性**：单纯通过提示“创造信号”是不够的，如果提示太直接，模型在无提示测试时会失效。HiLL 成功将提示依赖度保持在较低水平。
*   **提示演化**：随着训练进行，提示器学会了生成更短、更具概念性（Conceptual）的提示，而不是直接设置代数方程。
*   **动态适配**：HiLL 能够随着推理器能力的增强，自动将重心转移到新的“能力边界”题目上。

### 7. 优点
*   **理论支撑**：推导了迁移边界（Transfer Bound）理论，为提示依赖度提供了数学依据。
*   **端到端闭环**：实现了提示器与推理器的协同进化，解决了静态提示无法适配模型进步的问题。
*   **启发式引导**：通过奖励机制迫使模型生成“授人以渔”而非“授人以鱼”的提示，提升了模型底层的推理素质。

### 8. 不足与局限
*   **计算开销**：由于需要在线生成提示并进行二次采样，且涉及两个模型的联合训练，其训练耗时显著高于标准 GRPO（2.6x - 3.8x）。
*   **依赖参考答案**：提示器在训练阶段仍需依赖 ground-truth 解决方案来生成提示，这限制了其在完全无监督或无参考答案场景下的应用。
*   **模型规模限制**：实验主要集中在 3B 和 7B 模型，尚未在大规模模型（如 70B+）上验证其扩展性（Scaling Law）。
